# 0002. 音は「DSP生成した音ファイル」を同梱する（リアルタイム合成をやめる）

- Status: Accepted
- Date: 2026-06-03

## Context

初期実装は効果音・BGMを Web Audio の素朴なオシレーター（矩形/三角波）でその場合成していた。
結果は「ショボい」。リアルな木の着石音や上質なベル・残響は、素朴なリアルタイム合成では
品質の天井が低く、高級路線（緑＋金・明朝UI・3D盤）の世界観と噛み合わない。

検討した代替：
- リアルタイム合成を作り込む（多層＋残響）：ファイル不要だが、リアルな木の打音は本物に届かない。
- CC0フリー音源を同梱：実録音で最もリアルだが、素材探しとライセンス確認の手間。
- 自前でDSP生成した音ファイルを同梱：フィルタ済みノイズ＋共鳴で打音、倍音合成＋減衰でベル、
  和音パッド＋残響でアンビエント。ライセンス確認不要、品質はリアルタイム合成より大きく上。

当初は「音源ファイル0・全合成」を制約にしていたが、これは8bit路線時の方針であり、
高級リアル路線に転換した時点で見直す。

## Decision

効果音・BGMは、オフラインでじっくりDSP生成した音ファイル（WAV/MP3）を同梱する。
- 着石＝木の打音、返し＝連鎖で音程上昇、BGM＝静かなアンビエント（形勢3段階で明暗）。
- ベル／ファンファーレ等の演出音もファイル化。
- BGMと効果音は別々にミュート可。演出音は効果音チャンネルに乗る。
- service worker でキャッシュしてオフライン起動を維持する。

## Consequences

- 良い面：音の品質が大きく上がり、世界観が一貫する。
- コスト：音ファイルぶんの容量増（数百KB）。生成スクリプトを保守する必要がある。
- 「ファイル0」制約は放棄。オフライン性は service worker のキャッシュで担保する。
