# CONTEXT — リバーシ 用語集

価値定義（最上流の確定事項）と、このプロジェクトで使う言葉の定義。実装の詳細は書かない。

## 価値定義（確定済み）

2026-06-10 の value-grill により確定。確定済み事項であり、後続の grill・実装はこれを前提として扱い蒸し返さない。

### 定義1：誰のため＋欲望

- ターゲットは製作者自身ただ一人——飽きっぽく、1手ごとの快に貪欲な自分。
- 切り捨てた層：家族・友人などの対戦相手（気持ちよさの審査員からは除外）／Z世代・一般ユーザー（届ける経路を持たない）／シンプル・サクサク志向層。
- 知人・家族は対戦相手・戦績記録の対象であってターゲットではない。
- 欲望の根：「1手1手、常に気持ちいい」という直接体験。
- 収益化はこのアプリのゴールではない。
- 成功条件：私が1手1手常に気持ちいいと感じながらプレイし続けられること。測定器＝外発的継続装置（段位・デイリー報酬等）ゼロの状態で、私が自発的に遊び続けているという行動事実。
- 設計仮説：継続は目的でなく結果。1手の快が継続を生む。快の源泉は演出の新奇性ではなく、打つ行為そのものの手応えに宿る——に賭ける。

### 定義2：どこまでやれば価値か

- PR（確定文）：「既存のオセロアプリに物足りなさを感じているあなたに朗報です。1手1手に手応えがあり、刺激を感じられる演出に拘ったリバーシのゲームを開発しました。ついついのめり込み、やめ時がわからなくなっている自分に気づくことでしょう。」
- 競合較正（2026-06-10、調査2本）：演出投資の現存最高到達点は Piccolo: Othello（上質ミニマル方向）。「素の1手の五感フィードバック」を追求した純オセロは日英市場で空白。
- 品質ラインの床（未満は失格）：タップ後100ms以内の視覚応答／全ての手に2層以上の音（操作音＋結果音）／めくりは逐次（同時めくりは不合格）／全アニメにイージング（線形は不合格）／Piccolo水準の滑らかさ。
- 主戦場（v1落第リスト対応）：素の1手の五感フィードバック——めくり1枚ごとの音＋ピッチ上昇（Balatro式）／重みはフォロースルーで出す（着石の沈み・裏返り後の微バウンド。予備動作の追加はしない）／石と盤の質感・物理感（Hearthstone式。石のテクスチャの高級感）／終局音の作り込み。
- 変動則：演出強度は手の価値に比例させる。スポット演出は大きい手限定を維持。
- 盤面明度：明度調整機能（スライダー）を設定に追加する。
- 完了条件：酔い対策（演出強度設定 or prefers-reduced-motion 対応）まで含めて完了。
- やらないこと（境界の外）：複数演出テーマの実装（切替UIは作るが切替先エフェクトは作らない）／ストア申請（ただしコンバートしやすい設計は維持）／ハプティクス（iOS Safari に正攻法なし）／スキップ・倍速モード（横槍裁定で却下）／外発的継続装置／CPUの強さの改善（品質審査の対象外）。
- 技術注記：アニメは delta-time ベース必須（低電力モードの30fpsスロットル対策）。

### 定義3：デザインプリンシプル

1. テンポ vs 重み → 重みを優先する（サクサク感を捨てる）。
2. 石・盤（物体）は高級感を優先する（派手さを捨てる）。演出（イベント）は気持ち良さ・派手さを優先する（上品さ・ミニマリズムを捨てる）。
3. 変動性を維持したまま、毎手の快の床を上げる（「全ての手に最大演出」を捨てる）。
4. 拡張点はUIまで作り、中身は作らない（決め打ちの純度を捨て、将来の演出切替の口を残す）。

横槍裁定の実例（2026-06-10）：「連戦すると溜めとめくり演出が冗長。スキップか倍速モードを」→ 却下。スキップしたくなる演出は設計の失敗であり、デフォルトの演出自体を調整して直す（原則1・3による裁き）。

## プレイヤー / 識別

- **プロフィール**：名前だけを持つ登録済みの人。上限10人（2026-06-04 に2人から拡張済み）。戦績の個人集計の対象。
- **ゲスト**：その対局限りの一時的な人。プロフィール枠を消費せず、個人集計の対象にもしない。
- **CPU**：コンピュータの対戦相手。難易度（かんたん/ふつう/つよい）を持つ。

（注：以前あった「主役（メインプレイヤー）」概念は廃止。BGMは局面フェーズ、演出は盤全体のグローバルなイベントで決まり、特定プレイヤー視点には依らない。）

## 対局

- **先攻 / 後攻**：先攻＝黒、後攻＝白。黒が先に打つ。
- **手番**：今打つ番の色。
- **パス**：打てる場所が無い手番が、着手せず相手に番を渡すこと。
- **待った**：直前の局面に戻す操作。回数制限なし。
- **棋譜**：1対局の全手順を標準表記（列 a–h／行 1–8、パスは ps）で並べた記録。
- **連鎖めくり**：着手で返る石を、一度に変えず置いた石から外側へ波状に1枚ずつ裏返す見せ方。

## 形勢・演出

- **形勢**：ある色から見た有利不利。序中盤は盤上の位置価値、終盤は石数差を重視して判定する（ハイブリッド）。主に「逆転」（リードしている色が入れ替わる瞬間）の判定や、終盤の競り合い度の判定に使う。特定プレイヤー視点には固定しない。
- **局面フェーズ**：BGM切替の段階。2状態＝「通常（前半＋中盤）」と「終盤」。盤の埋まり具合（石数・空きマス）で決まり、特定プレイヤー視点に依らない。終盤は緊迫感のある音のみ（接戦/一方的の分岐は廃止）。終盤曲は道化師のギャロップ風の暗めの疾走ギャロップ（短調・ベル無し）。
- **スポット演出**：特定の瞬間に出す光と音の演出。対象＝大量返し／角取り／逆転（リードが入れ替わる）／終局（勝者へのファンファーレ）／完封（一方が0枚）／パス／ラス1マス。すべて盤全体のグローバルなイベントとして発火し、特定プレイヤー視点には依らない。浮遊テキストは出さない（勝敗などの文字情報は操作画面側で表示する）。
- **着手の溜め**：石を置くとき、盤面の真上に石が現れて空中で一瞬静止してから落ちる予備動作。「構えて打つ」ためのアンティシペーション。
- **めくりの溜め**：返る各方向の先頭石（置石に隣接する1枚目）が、回転せず水平に浮いて一瞬止まり、全方向そろって返る放射状の見せ方。連鎖めくりの号砲にあたる。
- **ヒットストップ**：四隅に着手した瞬間に盤全体の進行を一瞬止め、置いた角石を小刻みに震わせ、光と音を同じ一点に重ねる決め演出。スマブラのヒット停止に着想。止まりが収まってからめくり連鎖が始まる。

## 記録の可否

- 登録ユーザー（プロフィール）が1人でも対局に参加していれば、その対局を記録する。
- ゲスト同士、および登録ユーザーが居ない組み合わせ（ゲスト対CPU等）は記録しない。

## 設定

- **設定**：対局に依らないアプリ全体の好み。次回起動時も保持される（永続化）。歯車アイコンからいつでも開ける。以下を持つ：
  - **BGM音量 / 効果音音量**：それぞれ 0〜100% の連続値。100% が既定の聞こえ方。
  - **BGMミュート / 効果音ミュート**：音量とは独立した一時消音。解除すると元の音量に戻る。
  - **エフェクト演出の表示可否**：OFFにすると[スポット演出](#形勢演出)（特別な瞬間の光・決め演出。ヒットストップ含む）を出さない。石を置く・めくる基本アニメと、文字での告知（パス等）は残す。音は効果音側の設定に従う。
